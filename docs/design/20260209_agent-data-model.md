The data returned by the tools (often shell commands) are very huge, multiple kb+ or even extreme case, MB of data. If all of this is saved to the workflow, it can pollute very quick (and with the 4M of limits for gRPC, it is even more critical).

But if we outsource these content to a content-addressable storage, it can accelerate the replay, but we still need to fetch it during replay if the content of the task is used for computed (like parsing and read some fields from there), so it still ends up with an issue (less critical). A potential direction is to only pass by reference to the data for large data.
